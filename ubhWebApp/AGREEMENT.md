# UBH Frontend Team Agreement 🐱💻✨

Hey team!

This is our written agreement for our frontend team.

For time effiency, I won't require approval before every merge because sometimes that takes too much time.

But (_very important!_)

**IF OUR `main` BRANCH BREAKS BECAUSE OF SOMEONE'S CODE, THAT PERSON WILL BE RESPONSIBLE FOR PAYING FOR A MEAL FOR ALL TEAM MEMBERS!**

This is a ladder deal:

- For the first time, the total meal cost should be at least **$100** for the 4 of us.
- If the same person repeatedly breaks the `main` branch, the total cost will go up by **50%** each time.

Hehehe 😈

So, for the sake of your bank account:

- Do not push _FAULTY_ code.
- Do not _DELETE/ACCIDENTALLY OVERWRITE_ someone else’s code.
- Do not push directly to `main`.
- Always branch out to your own branch.
- Always run and check your code before merging.
- Always `git pull` from `main` regularly.

Pulling from `main` helps avoid Git conflicts and prevents accidentally overwriting someone else’s work.

For Git practice and workflow rules, please refer to the `GUIDE.md` file in this repository.

### Optional Merge Approval ✅

If you are unsure about your work, you can request approval from someone on the team before merging into `main`.

### Emergency Save Rule 🛟

If you notice that your code broke the `main` branch, but you fix it before anyone notices and fully recover everything back to normal, you will be exempt.

We will count it as if it never happened. 👍

## Final Reminder

Keep the `main` branch working.  
Protect your wallet. 😄
